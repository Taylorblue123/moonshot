## 1. Objective

Render the **first visible scene** of Moonshot: a layered, painterly, lo-fi dusk
landscape that recognizably matches the reference (`image.png` — a grass hill
with blue flowers, a black fence along the ridge, a giant cumulus sky, a crescent
moon, a lone character silhouette, and drifting blue butterflies).

**Why this milestone exists:** M1 is the foundation for the whole product. Its
real job is to prove that the scene can be rendered **and** that every part of its
look is driven by configuration data — because that configuration surface is what
the AI agent will later control from chat (M2 exposes it as an API, M4 turns it
into the agent's tool schema). If M1 hardcodes its look, the entire agent premise
breaks. So M1 has two intertwined goals:

1. **It looks like the reference** — a believable, atmospheric dusk scene with
   gentle life and a finished "mood."
2. **It is fully parameter-driven** — the scene's appearance is determined by a
   single configuration object, with no tunable values baked into the rendering.

---

## 2. Definition of Done

M1 is done when, running the app locally, a reviewer sees and can verify all of:

1. **A layered scene with depth** — sky, clouds, the fence + hill ridge, and a
   grass field are all visible and clearly sit at different depths.
2. **Gentle life (motion):** clouds drift, grass sways, and butterflies flutter.
   Nothing is a static still frame. (The camera itself is fixed — all motion comes
   from the content, matching the static reference composition.)
3. **A finished mood:** a visible overall color grade and vignette unify the
   frame into the lo-fi dusk feel of the reference (not a raw, untreated render).
4. **Fully parameter-driven:** the scene's entire look is read from a single
   configuration object. A reviewer can change one value (e.g. wind strength or
   vignette amount) in one place, reload, and immediately see the scene respond —
   with no tunable magic numbers hidden in the rendering code.
5. **Recognizably the reference:** side by side with `image.png`, a viewer would
   say "that's the same scene / same vibe."
6. **Healthy:** runs at a smooth frame rate on the dev machine with no errors.

**The bar is "recognizable, not pixel-perfect."** Matching the reference's *vibe*
— palette, layout, atmosphere, motion — is success. Exact reproduction is not
required and is explicitly not worth chasing in M1.

---

## 3. Non-Goals (explicitly out of scope for M1)

- **No control API or debug panel** — exposing the parameter object as callable
  setters is **M2**. M1 only requires that the scene *is* parameter-driven.
- **No audio** — that's **M3**.
- **No agent and no chat** — that's **M4+**.
- **No weather systems** (rain, snow, lightning, fog). Only the clear-dusk look is
  implemented in M1. Later weather may be anticipated in the configuration shape,
  but M1 does not implement it.
- **No streaming / OBS / YouTube** — that's **M6+**.
- **Not pixel-perfect** to the reference (see DoD).
