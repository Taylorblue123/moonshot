## 1. Goal

Render a **2.5D layered painterly scene** in the browser that recognizably matches
the reference (Antent *"end of era / ambient mix"* — a grass hill, a black fence
along the ridge, a giant cumulus sky, a lone character silhouette, drifting blue
butterflies, in a lo-fi dusk palette). It must have **gentle life** (drifting
clouds, swaying grass, fluttering butterflies, subtle parallax) and a **post-FX
mood pass**.

M1 is the foundation for the whole product. Its real job is to **prove the
rendering pipeline** — textured quad → custom GLSL shader → composited layers →
post-processing pass → screen — and to establish the **`params`-driven
architecture** that M2's control API and M4's agent tool schema will plug into.

### Non-goals (explicitly out of scope for M1)
- No control API / debug panel (that's **M2**).
- No audio (that's **M3**).
- No agent, no chat (that's **M4+**).
- No weather/rain/snow systems yet (the *hooks* exist via `params`, but only a
  clear-dusk look is implemented). Rain etc. come as polish after the core layers.
- No streaming/OBS (that's **M6**).
- Not pixel-perfect to the reference — "recognizably the same vibe" is the bar.

---

## 2. Definition of Done

Open `localhost` (via `npm run dev`) and see:

1. A 2.5D dusk scene with these layers visible in depth order: sky gradient →
   clouds → fence + hill line → grass → foreground grass + butterflies.
2. **Motion:** clouds drift, grass sways, butterflies flutter, the camera does a
   slow parallax drift.
3. A **post-processing pass** is applied (visible color grade + vignette).
4. **Every tunable value is read from a single `params` object** — changing a
   value in `params.js` (and hot-reloading) changes the scene. No tunable magic
   numbers buried in shaders/meshes.
5. Runs at a smooth frame rate on the dev machine; no console errors.

A reviewer should be able to tweak, say, `params.wind.strength` or
`params.mood.vignette` in one file and immediately see the effect.

## 3. Tech & Conventions

- **Stack:** Vite + Three.js + **JavaScript** (not TypeScript).
- **Types:** use **JSDoc typedefs** (`/** @typedef ... */`) on the `params` shape
  for editor autocomplete without a compile step.
- **Three.js role:** used as a **2.5D compositor** — textured `PlaneGeometry`
  quads + custom `ShaderMaterial`, a single camera, and `EffectComposer` for
  post-FX. We do NOT use 3D lighting/shadows/PBR.
- **Camera:** gentle perspective (~40° FOV) OR orthographic — see §6. Default:
  gentle perspective for subtle depth.
- **Art source:** **procedural-first** (shaders generate sky/clouds/grass), so the
  scene is fully parametric for the agent. Illustrated PNG assets may replace
  individual layers later (hybrid is fine); not required for M1.
- **Architecture rule (non-negotiable):** all tunable state lives in `params`;
  modules read from it each frame. This makes M2 (control API) a thin set of
  setters over `params`, not a refactor.

```
moonshot/

├── package.json            # three + vite (exists)

    │   ├── sky.js          # sky gradient quad (ShaderMaterial)
    │   ├── stars.js        # stars (points) + moon sprite + shooting stars
    │   ├── clouds.js       # cumulus band (noise shader or texture), drifts by wind
    │   ├── ridge.js        # fence + hill silhouette (mostly static plane)
    │   ├── grass.js        # mid + foreground grass, vertex-sway by wind
    │   ├── butterflies.js  # instanced butterflies, flutter + drift
    │   └── character.js    # lone silhouette (sprite); optional in M1
    ├── postfx/
    │   ├── composer.js     # EffectComposer + RenderPass + lofiPass
    │   └── lofiPass.js     # custom ShaderPass: color grade + vignette (+ grain)
    └── shaders/            # .glsl or template-literal strings, per layer
        ├── sky.frag / sky.vert
        ├── clouds.frag
        ├── grass.vert / grass.frag
        └── lofi.frag
```

> Keep each layer in its own module exposing a small interface (see §7) so the
> scene is composable and each piece is independently testable/visible.

## 5. Layer Stack (far → near)

| z (far→near) | Layer | Look | Animated by |
|---|---|---|---|
| 1 | **Sky** | vertical gradient (dusk blue → pale near horizon) | `params.timeOfDay` (gradient stops, tint) |
| 2 | **Stars + moon** | sparse points + a crescent; faint by day | `starTwinkleSpeed`, `shootingStarRate` |
| 3 | **Clouds** | painterly cumulus band; soft, billowy | `uWind` (drift), `cloudCoverage`, `cloudDriftSpeed` |
| 4 | **Ridge (fence + hill)** | dark silhouette line across the frame | mostly static; tiny parallax |
| 5 | **Grass (mid)** | textured/shaded grass field | `uWind` vertex sway |
| 6 | **Foreground grass + butterflies** | taller grass + drifting blue butterflies | `uWind` sway + butterfly motion |
| — | **Post FX** | color grade + vignette over the whole frame | `params.mood.*` |

Depth is created three ways, combined: (a) real `z` separation of the quads,
(b) different **parallax factors** per layer (far moves least), (c) painterly
overlap + the dark foreground framing.

## 6. Key Design Decisions

### Camera: gentle perspective vs orthographic
- **Gentle perspective (~40° FOV) [default]:** real depth, cozy photographic feel,
  parallax reads richly. Matches the reference.
- **Orthographic:** flatter "diorama" look; choose only if we later want a
  paper-craft aesthetic.
- This is one line to swap, and the camera config lives in `params.camera`, so the
  choice is reversible at no cost. Start with gentle perspective.

### Parallax model
Camera drifts slowly (sinusoidal) within a small range; each layer is offset by a
per-layer `parallaxFactor` (sky ≈ 0.0, clouds ≈ 0.2, grass ≈ 0.8, foreground ≈
1.0). Driven by `params.camera.parallaxAmount` / `parallaxSpeed`.

### Broadcast params
`wind` and `timeOfDay` are **broadcast** values: one number in `params`, consumed
by many layers (wind → clouds + grass + butterflies; timeOfDay → sky + stars +
mood). Layers subscribe by reading `params` each frame. This is the abstraction
the agent will love ("make it windy" = one knob, whole scene responds).

### Procedural vs illustrated art
Procedural-first (shaders). If a layer is too hard to make pretty procedurally
(e.g. the cumulus clouds), it may use a painted texture instead — the layer's
external interface (`update(params, dt)`) stays the same either way.

## 7. Module Contracts

Each layer module exports a factory returning an object with a consistent shape,
so `scene.js` can build/update them uniformly:

```js
// layers/<name>.js
/**
 * @param {THREE.Scene} scene
 * @param {SceneParams} params
 * @returns {{ object3d: THREE.Object3D, update(params: SceneParams, dt: number, t: number): void, dispose(): void }}
 */
export function createSkyLayer(scene, params) { /* ... */ }
```

- `object3d` — added to the scene by `scene.js`.
- `update(params, dt, t)` — called every frame; reads `params`, advances animation
  (`dt` = delta seconds, `t` = elapsed seconds), updates uniforms.
- `dispose()` — frees geometry/material/textures (good hygiene for a 24/7 app).

`scene.js` exports `createScene(params)` returning
`{ scene, camera, update(params, dt, t) }` where its `update` fans out to all
layers + camera parallax.

`postfx/composer.js` exports `createComposer(renderer, scene, camera, params)`
returning `{ composer, update(params), setSize(w,h) }`.

## 8. `params` Object (M1 subset)

M1 implements the **clear-dusk** subset of the full parameter list (full list in
`notes.md` §8 / `notes.html` §8). Unimplemented params may exist in the object as
hooks but need not affect the scene yet.

```js
/** @typedef {Object} SceneParams ... (full typedef in params.js) */
export const params = {
  timeOfDay: 18.5,                 // dusk (hours, 0–24)  ★broadcast
  sky:    { topColor: '#1f6fd0', horizonColor: '#bfe0ff' },
  stars:  { density: 0.4, twinkleSpeed: 1.0, shootingStarRate: 0.03 },
  clouds: { coverage: 0.6, driftSpeed: 0.02, driftDir: 1, tint: '#ffffff', height: 0.55 },
  wind:   { strength: 0.4, direction: 1 },        // ★broadcast
  grass:  { color: '#243b3a', lushness: 0.7, height: 1.0 },
  butterflies: { count: 40, speed: 0.5, color: '#8fb7ff', behavior: 'calm' },
  camera: { fov: 40, parallaxAmount: 0.15, parallaxSpeed: 0.05, zoom: 1.0 },
  mood:   { colorTemperature: 0.45, brightness: 1.0, contrast: 1.05,
            saturation: 1.0, vignette: 0.35, bloom: 0.2, filmGrain: 0.08 },
  // hooks for later milestones (not implemented in M1):
  weather: { precipType: 'none', precipIntensity: 0, fogDensity: 0.1, lightningRate: 0 },
};
```

## 9. Build Order (each sub-step is runnable + visibly verifiable)

Teaching note: each step maps to a WebGL→Three.js concept; the user knows
WebGL/GLSL but is new to Three.js, so explain the mapping at each step. Pace:
**step-by-step, file by file**, pausing for the user to run after each.

| Step | What | Concept (WebGL → Three.js) | Visible result |
|---|---|---|---|
| **M1.0** | Scaffold + `npm install`; renderer + render loop in `main.js` | `gl` context + `requestAnimationFrame` → `WebGLRenderer` + `setAnimationLoop` | Cleared canvas (dusk color) |
| **M1.1** | `params.js`; **sky** gradient quad | VBO + frag shader → `PlaneGeometry` + `ShaderMaterial` | Full-screen sky gradient |
| **M1.2** | Camera + parallax drift in `scene.js` | view/projection matrices → `PerspectiveCamera` | Sky drifts subtly |
| **M1.3** | **Clouds** layer + `uWind` | another shaded quad; broadcast uniform | Cumulus band drifting |
| **M1.4** | **Ridge** (fence + hill) + **grass** with vertex sway | vertex shader displacement | Hill/fence + swaying grass |
| **M1.5** | **Butterflies** (instanced) | instanced draw → `InstancedMesh` | Butterflies fluttering |
| **M1.6** | **Stars + moon** + **character** sprite | points / sprites | Night detail + lone figure |
| **M1.7** | **Post-FX** pass (`lofiPass`) | FBO + fullscreen blit → `EffectComposer` + `ShaderPass` | Graded, vignetted mood (home turf) |
| **M1.8** | Wire everything through `params`; tidy idle motion | — | DoD met: tweak `params` → scene changes |

Get the pipeline proven on **M1.0–M1.1** (one layer, one shader) before adding the
rest. If a layer resists looking good, ship it "good enough" and move on — polish
is post-M1.

## 10. Risks & Mitigations

| Risk | Mitigation |
|---|---|
| Shader/mood polish rabbit-hole | Time-box per layer; "recognizable, not perfect" is the bar; polish is post-M1 |
| Procedural clouds look bad | Fall back to a painted cloud texture for that layer (interface unchanged) |
| Hardcoding values now → M2 refactor later | Enforce `params`-driven from M1.1; review DoD #4 |
| Scope creep (adding rain/audio/agent) | Those are M3–M5; M1 only hooks them in `params`, doesn't implement |
| 24/7 perf later | Note only; M1 just shouldn't leak (use `dispose()`); real perf work is M6/M8 |

## 11. Handoff to M2

When M1 is done, M2 builds the control API + debug panel. Because everything is
`params`-driven, M2 is: (1) expose setters/getters over `params`
(`window.scene.set('wind.strength', 0.8)`), (2) a small dat.GUI/lil-gui panel bound
to `params`, (3) ensure each setter triggers the relevant layer to pick up the
change. The setter surface defined here becomes the **tool schema** the M4 agent
calls — so name params clearly and keep the shape stable.
